Instrucciones: Desarrolle el proyecto final utilizando React + Vite para el frontend. El backend puede implementarse en Python Flask o Java Spring Boot. El sistema debe ser modular y cubrir las funcionalidades descritas. La entrega debe incluir: 1) Código fuente organizado, 2) Capturas de pantalla de ejecución 3) Documentación (README).
Fecha. 14-07-26
Desarrollar un proyecto de un sistema académico integral, implementado con frontend en React + Vite y backend en Python Flask o Java Spring Boot, que permita gestionar de manera segura y eficiente los procesos académicos de facultades y especialidades. El sistema debe incluir módulos de matrícula, cursos y docentes, control de notas, record académico, certificados y documentos, así como administración y seguridad con roles diferenciados para estudiantes, docentes, administradores y dirección. El objetivo es construir una aplicación funcional, modular y escalable que refleje las necesidades reales de una institución académica y demuestre buenas prácticas de programación y trabajo colaborativo.
Módulos a Implementar
Módulo de Matrícula
Estudiante: Solicita matrícula, descarga ficha.
Administrador: Valida requisitos, registra pagos, genera ficha oficial.
Dirección: Supervisa estadísticas de matrícula.
Módulo de Cursos y Docentes
Docente: Visualiza cursos asignados, carga sílabos y notas.
Administrador: Asigna docentes y gestiona horarios.
Dirección: Evalúa carga docente y cumplimiento del plan de estudios.
Módulo de Notas
Docente: Registra notas parciales y finales.
Estudiante: Consulta hoja de notas por ciclo.
Administrador: Valida actas y consolida notas.
Dirección: Supervisa indicadores académicos.
Módulo de Record Académico
Estudiante: Accede a su historial completo.
Administrador: Genera reportes consolidados.
Dirección: Analiza desempeño por cohorte o programa.
Módulo de Certificados y Documentos
Estudiante: Solicita certificados y constancias en línea.

Administrador: Emite certificados con firma digital/QR.
Dirección: Autoriza emisión de documentos oficiales.
Módulo de Administración y Seguridad
Administrador: Define perfiles de acceso (roles).
Dirección: Controla auditorías y reportes estratégicos.
Todos los usuarios: Acceden según permisos asignados.
Evaluación (0–20 puntos)


Entregables
Proyecto con frontend y backend.
Documentación con Capturas de pantalla de las funcionalidades principales.
Presentación y exposición final
